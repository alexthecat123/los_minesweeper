; =============================================================================
;  MINESWEEPER-ALERTS.TEXT -- the phrase-file SOURCE for Minesweeper.
;
;  Compiled by the Workshop 'Alert' tool into the binary <tool>PHRASE file
;  that InitMP reads at run time (see MINESWEEPER-MAKE.TEXT).  Every string
;  the user ever sees -- alerts, menus, dialog labels -- lives here, none in
;  the Pascal source: ship a different phrase file, get a different language,
;  same executable (06-lib-am).
;
;  FORMAT RULES THAT BITE (06-lib-am, "The phrase-file source format"):
;    - a comment must occupy its OWN line (a trailing ';' on a data line
;      becomes part of the data);
;    - blank lines are STRUCTURAL: one ends the button block, one ends each
;      alert's text, one ends each menu block -- do not add extras;
;    - alert numbers must be strictly ascending; #1 and #2 are REQUIRED;
;    - the alert section and the menu section each end with a lone 0.
;
;  The numbers here must stay in lockstep with two things in the Pascal:
;    - alert / phrase constants in MINESWEEPER-GLOBALS.TEXT (alrt*/ph*);
;    - menu ids and item positions in MINESWEEPER-GLOBALS.TEXT (m*/mi*) --
;      remember a '-' separator line COUNTS as an item position.
; =============================================================================
;
; ---- FILE HEADER ------------------------------------------------------------
; Version stamp: any integer; compared at run time against alertVerNum (= 2)
; in MSGlobals -- a mismatch is only warning -3076.  Bump both together.
2
; cacheSize: resident alert-text slots, and cacheBytes: the cache's heap byte
; budget.  InitMP reads these two lines and passes them to InitAlerts.
; *** ESTIMATES.  The Alert tool prints suggested InitAlerts(slots, bytes,..)
; *** values when it compiles this file; on the first real build, paste those
; *** numbers over these two lines.  Chosen generously: 6 slots / 3000 bytes
; *** comfortably covers our longest alert plus the always-cached alert 2.
6
3000

; ==== ALERTS =================================================================
;
; -- #1 REQUIRED: the generic stand-in shown when code asks for an alert
;    number that is not in this file.
1 note alert
Minesweeper could not find the message that should appear here.  If you are
unable to proceed, contact a qualified service representative, and mention
the number ^N.

; -- #2 REQUIRED: the "Swan Song", locked permanently in cache slot 0 so it
;    can still be shown when disk or memory has failed (06-lib-am ReadAlerts).
;    Wording modelled on the Calculator's (aphp-T12ALERT.TEXT).
2 stop alert
Minesweeper is having technical difficulties accessing the disk that contains
the Minesweeper tool.^L^L
Put away your documents one at a time, or push the on-off button to save
them all.

; -- New game while one is in progress ('caution ok': question icon, OK is
;    the default button).  CautionAlert returns true for OK.
10 caution ok alert
Start a new game?  The game in progress will be lost.

; -- The win notice.  ^1 = mine count, ^2 = elapsed seconds, both filled by
;    ParamAlert just before the NoteAlert (MINESWEEPER-MAIN.TEXT, DoWin).
20 note alert
You swept all ^1 mines in ^2 seconds.

; -- Best Times.  ^1/^2/^3 are one preformatted line per fixed level
;    ("012  Alex", or phrase 915's text when a level has no record yet).
30 note alert
The best times so far!!!^L^L
Beginner:  ^1^L
Intermediate:  ^2^L
Expert:  ^3

; -- Reset Best Times ('caution cancel': CANCEL is the default button, so a
;    reflexive Return keeps the records; destroying them takes a deliberate
;    click).
40 caution cancel alert
Reset all best times?  The record book will be emptied.

; -- Custom-value validation.  TWO-STAGE by design: the first offence is only
;    a soft beep (the caret is already back on the bad field); a repeat gets
;    a medium beep AND this box.  ^1 = the text typed, ^2 = the field's own
;    label (phrases 910-912), ^3/^4 = the legal bounds -- ^1..^3 via
;    ParamAlert, ^4 via ArgAlert(4,..) (06-lib-am).
50 stop soft; medium alert
"^1" is not a valid ^2.  It must be a whole number between ^3 and ^4.

; ==== GET PHRASES ============================================================
; Number-only headers: stageless, exactly ONE verbatim text line each, no
; escape expansion; fetched with GetAlert (06-lib-am).  901-909 are reserved
; for DTAlert, so the visible ones start at 910.
; Each still needs its OWN terminating blank line: AlertGen reads text lines
; until a blank one, and a comment line does NOT end the text -- without the
; blank, the next phrase's number is read as a second text line and the tool
; stops with "Only one line of text allowed when no severity".
;
; -- 900: the Field Editor "word triples" for double-click word selection
;    (FEInit; 06-lib-fe 'qualifies').  Groups of four: <left><this><right>
;    plus a space; 'x' = any, 'A' = alpha, '9' = digit.  This is the
;    Calculator's own table (APHP's Menu2Buzz item) minus its last two
;    triples, which use high-bit Mac Roman characters (curly quote and
;    dash variants) that do not survive this archive's encoding; their
;    absence only means those two exotic characters do not glue words
;    together on double-click.
900
xAx x9x x$x x%x x.9 x.% 9,9 A'A A-A A.A
